Authentication policy

Deployment-wide — applies to all human principals. Editable by admin/root.

Require MFA (AAL2)
NIST 800-63B AAL2 — every human session must complete a second factor.
Re-authentication window
Force re-auth during an extended session regardless of activity (≤30d @AAL1, ≤12h @AAL3).
Idle timeout
End a session after inactivity (AAL3 requires ≤15 minutes).
Brute-force protection
Soft lock with increasing delay; ≤100 failed attempts/hour/account (OWASP V6). Reset never locks an account.
enforced

Agents don't have sessions — they authenticate per request with an API key. Manage those under Access Control and NHI Access Review.
