What is ContextHub?

ContextHub is a self-hosted persistent memory and guardrails system for AI agents. It stores lessons, decisions, workarounds, and preferences that agents can query via MCP tools. Unlike cloud-hosted solutions, all data stays on your infrastructure.

How do I connect my AI agent?

Point your MCP-compatible agent to http://localhost:3000/mcp and pass project_id in your tool calls.

The server exposes tools like search_lessons, check_guardrails, and add_lesson.

What storage backend does it use?

PostgreSQL with pgvector for embeddings storage and vector similarity search. The knowledge graph uses Neo4j. Data is persisted via Docker volumes.

Can I use it with multiple projects?

Yes. Each project has its own isolated namespace. You can also create project groups to share knowledge across related repositories. Projects in the same group can connect and search each other's lessons when the "Include group knowledge" toggle is enabled.

How does the tiered code search work?

Code search uses three tiers in order: exact match (fastest, filename/symbol lookup), glob pattern (file path patterns), and semantic search (embedding-based similarity). Results are merged and deduplicated, with the highest-confidence tier taking priority.

What models are supported?

Any OpenAI-compatible API endpoint works — LM Studio, Ollama, vLLM, OpenRouter, or the OpenAI API itself. Configure providers in the Model Providers settings page and assign models to specific features (embeddings, distillation, chat, etc.).
