Settings Access Control

Access Control

A credential authenticates to a principal. The principal — not the key — carries authority, through its grants.

What changed (foundation F1): a key no longer carries a free-floating role + project string. It now binds to a principal, and authority comes from that principal's grants. The old built-in roles (admin/writer/reader) map to seed grants on the bound principal — same effect, but now auditable and revocable per-scope.
