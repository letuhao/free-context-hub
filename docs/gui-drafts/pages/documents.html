1. RESTful Conventions

All endpoints must follow REST conventions. Resource names should be plural nouns, and nested resources should reflect ownership hierarchy.

1.1 HTTP Methods

  • GET for retrieval (never mutate state)
  • POST for creation
  • PUT/PATCH for updates
  • DELETE for removal (must be idempotent)

2. Error Handling

All errors return JSON with error_code and message fields. HTTP status codes must match the error category.

{
  "error_code": "VALIDATION_FAILED",
  "message": "Field 'email' is required",
  "details": {
    "field": "email",
    "constraint": "required"
  }
}

3. Versioning

API versioning uses URL path prefix (/v1/, /v2/). Breaking changes require a major version bump. Non-breaking additions (new optional fields) are allowed within the same version.
