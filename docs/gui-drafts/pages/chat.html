Key Architectural Decisions

Here are the 5 foundational architectural decisions for the project:

  1. SQLite with WAL mode -- chosen for single-file deployment and concurrent read support. Configuration lives in src/services/lessons.ts.
  2. MCP as the primary interface -- allows any AI agent to connect without custom integration. The server exposes tools via the standard MCP protocol.
  3. Tiered code search -- exact match first, then glob, then semantic to balance speed and recall.
  4. Job queue with dead-letter -- failed jobs are preserved for debugging rather than silently dropped.
  5. Batch embedding calls -- reduces LLM round-trips for large ingestion operations.

The retry logic is in src/lib/retry.ts. Here is the core function:

typescript
export async function retry<T>(
fn: () => Promise<T>,
opts: RetryOptions = retryConfig
): Promise<T> {
let lastError: Error;
for (let i = 0; i < opts.maxAttempts; i++) {
try {
return await fn();
} catch (err) {
lastError = err as Error;
if (!isRetryable(err, opts)) throw err;
const delay = getBackoff(i, opts);
logger.warn(`Retry ${i+1}/${opts.maxAttempts}`);
await sleep(delay);
}
}
throw lastError!;
}

Key behaviors:

  • Uses exponential backoff with jitter
  • Only retries errors in the retryableErrors list
  • Logs each retry at warn level for observability
