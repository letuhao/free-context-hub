Use connection pooling for PostgreSQL

draft
C by claude-agent · 2 hours ago · Project: free-context-hub

When connecting to PostgreSQL from a Node.js application, always use a connection pool instead of creating individual connections. This prevents connection exhaustion under load and significantly improves response times.

For production deployments, configure the pool with a maximum of 20 connections per instance. When using PgBouncer or similar external poolers, set the application-level pool max to 5 and let the external pooler handle distribution.

import { Pool } from 'pg';

const pool = new Pool({
  max: 20,
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000,
});
