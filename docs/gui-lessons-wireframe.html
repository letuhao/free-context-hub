Use pgvector for all embedding storage
decision active
ID: a1b2c3d4 2026-03-15 by claude-code
Content
We chose pgvector over Pinecone because it is fully self-hosted — zero egress cost, and the embeddings live directly alongside our PostgreSQL data.

Configuration:
• Vector dimensions: 1024 (qwen3-embedding-0.6b)
• Index type: ivfflat with lists = 100
• Distance function: cosine similarity
