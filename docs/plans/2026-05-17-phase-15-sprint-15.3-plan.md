## Execution mode

**BUILD (T1–T8) is dispatched to a single fresh BUILD agent** — the Sprint 15.2 precedent.
The tasks form one dependency chain inside one service area (`migrations/` → `src/services`
→ `src/api` → `src/mcp` → wiring); a single agent holding the full design rev 3 is the right
unit. The agent does TDD per task and self-tests as it goes.

**VERIFY (T9) and the live smoke (T10) are run by the main session — fresh** — per the
Phase 6 evidence gate: the BUILD agent's own test runs are not the verification of record.

Task order is **strictly sequential T1 → T10** (each depends on the prior). TDD applies to
T2–T6 (tests RED → implementation GREEN → refactor). T1 (DDL), T7 (MCP thin adapters), T8
(wiring exports) are build-and-verify, not TDD.

## Code intent

**Every task's complete code intent is the referenced design section.** Design rev 3
§§0–7 are detailed, reviewed across 3 Adversary rounds, and final. This plan sequences the
work and pins each task's files + verification; it does **not** re-paraphrase the code
(re-paraphrasing risks drift from the reviewed spec). Implement exactly what the cited §
specifies — including §0.1 (the transaction/connection contract), §0.2 / §10 (the canonical
lock order `task → claim → request → request_step → artifact → topics`), and the rev-2/rev-3
fixes (self-decision guard, the full `artifact_versions` INSERT, the closed-topic pre-checks,
the `[0, 2147483647]` `weight` bound).

### T4 — `requests.ts` part B — `decideStep` + `resolveArtifact`
- **Files:** `src/services/requests.ts` (extend), `src/services/requests.test.ts` (extend)
- **Intent:** design §3.2 + §3.3 — `decideStep` (the `request` then `request_step`

  → `self_decision_forbidden` (`actor == submitted_by`) → `not_authorized`; the
  endorse / return / reject branches incl. next-step activation + `current_step` advance);
  `resolveArtifact(client, outcome, artifactId, actorId)` internal (the guarded
  `for_review → final|working` UPDATE `RETURNING version, content_ref`; the fully
