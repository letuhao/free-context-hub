## T2 — salience.ts gate + helper

**T2.1 — Add `isSalienceWriteDisabled()` exported helper.**
- Location: alongside existing `isSalienceDisabled` (around line 57).
- Implementation (direct-read, not cache):
  ```typescript
  /** True when writes to lesson_access_log are suppressed. Reads still work.
   *  Intended for "measurement isolation" — operators running QC baselines
   *  who want salience-aware ranking but NO accumulation of consideration-search
   *  rows that would pollute subsequent measurements (see goldenset-pollution
   *  friction class).
   *
   *  Reads process.env directly rather than going through getEnv() so that
   *  tests (and operators flipping the flag between baseline runs) see the
   *  change without a cache bust. The production code path recreates the
   *  container on env changes anyway, so cache-vs-direct is functionally
   *  equivalent at deployment time. */
  export function isSalienceWriteDisabled(): boolean {
    return parseBooleanEnv(process.env.LESSONS_SALIENCE_NO_WRITE) === true;
  }
  ```

**T2.2 — Add gate at `logLessonAccess` function top.**
- Insert as FIRST line inside the function body:
  ```typescript
  export async function logLessonAccess(pool: Pool, entries: AccessLogEntry[]): Promise<void> {
    if (isSalienceWriteDisabled()) return;  // NEW — 12.1e3
    if (entries.length === 0) return;
    // ... existing code unchanged
  }
  ```
