## TDD discipline

The two service modules are DB-integration code: each test file imports its module, so the
module must compile before its test runs. Order per service: write the service (T3/T5) →
write the test (T4/T6) → run the test; a failing test is the RED signal → fix the service
→ green. The migration (T1) and the wiring tasks (T2/T7/T9/T10) are not TDD — "just build
and verify" (CLAUDE.md Phase 5) — their gate is `tsc` + (for T1) an idempotent re-run.
