### Group B — Drafter via Sonnet subagent (no committed code; one-shot)

**Rationale:** drafting runs once during Sprint 16.1 and is never re-run. Building `genEvalDraft.ts` + env vars + template files would commit infrastructure for ephemeral work. Instead we spawn a Sonnet 4.6 subagent (via `Agent` tool with `model: "sonnet"` override) that reads queries.json + ground-truth sources and writes drafts back. Cleaner, faster, leaves zero archived code behind.

The subagent gets a self-contained prompt covering:
- exact JSON file paths to read/write
- which fields to add (`ideal_answer`, `must_contain_facts`, `answer_style`, `drafted_by='llm'`, `drafted_at`)
- which fields to preserve (existing retrieval fields untouched)
- the drafter instruction ("you see the GROUND-TRUTH sources — draft the ideal concise reference answer and list 3-7 atomic facts any correct answer must cover")
- the project-context primer (CLAUDE.md excerpt for tone/voice)

| # | Task | How | Verify |
|---|---|---|---|
| **B1** | Spawn Sonnet subagent for smoke pilot — draft 3 lesson queries only | `Agent({subagent_type:"general-purpose", model:"sonnet", prompt:"<self-contained drafter prompt>... draft 3 rows from qc/lessons-queries.json, write back inline, report diff"})` | manual: inspect 3 drafted rows; tune prompt if quality off; re-spawn if needed |

(B2/B3/B4 dropped — no `genEvalDraft.ts`, no `templates/drafter.txt`, no env vars. The subagent prompt is the template.)

## Risks & contingencies (Sprint 16.1 specific)

| Risk | Contingency |
|---|---|
| Drafter LLM produces bad atomic-fact lists (too vague, too specific, hallucinated) | B4 pilot catches this before C1-C7 batch. If output is bad → revise drafter prompt B3, re-run pilot. |
| Human review takes longer than 30s/row | OK — quality > speed. Spread C2/C4/C8 across multiple sessions if needed. |
| Edge-case authoring stalls on creativity | Reference: prior bug-hunt findings, existing lessons in DB, real chat history (13 msgs but readable). Inspiration sources documented in D1-D4 task notes. |
| Code-surface gen-eval rows feel awkward ("the answer is just the file") | Per DESIGN §5: chunks surface tells us if pure-retrieval-eval is enough for code. If reviewer concludes code gen-eval adds nothing — defer code rows to 16.1b; not blocking. |
| Validator catches schema violations during C/D batches | Fix the offending row(s) immediately; never commit with validator failing. |
