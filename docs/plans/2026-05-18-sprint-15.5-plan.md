### T2 — `src/services/intake.ts` — submitIntake, dismissIntake, getIntake, listIntake

**File:** `src/services/intake.ts`

Implement the four simple functions (no transaction needed — single-statement operations):

```
submitIntake({ project_id, topic_id?, kind, body, submitted_by }):
  1. Validate kind ∈ {'violation_report','suggestion','request'}; body non-empty; submitted_by non-empty.
  2. pool.query SELECT 1 FROM projects WHERE project_id=$1 → NOT_FOUND if missing.
  3. If topic_id: pool.query SELECT status FROM topics WHERE topic_id=$1 → NOT_FOUND or TOPIC_NOT_ACTIVE if not 'active'.
  4. pool.query INSERT INTO intake_items ... RETURNING * → row.
  5. If topic_id: appendEvent(pool, { topic_id, actor_id: submitted_by, type:'intake.received',
       subject_type:'intake', subject_id: intake_id, payload:{ project_id, kind, submitted_by } }).
  6. Return row.

dismissIntake(intake_id):
  1. pool.query UPDATE intake_items SET status='dismissed' WHERE intake_id=$1 AND status='received' RETURNING *.
  2. If 0 rows: re-SELECT to distinguish NOT_FOUND vs INTAKE_ALREADY_TRIAGED vs INTAKE_ALREADY_DISMISSED.
  3. No event. Return updated row.

getIntake(intake_id):
  1. pool.query SELECT * FROM intake_items WHERE intake_id=$1.
  2. NOT_FOUND if 0 rows. Return row.

listIntake(project_id, opts?):
  1. Build WHERE: project_id=$1 + optional kind=$N + optional status=$M.
  2. SELECT * FROM intake_items ... ORDER BY created_at DESC LIMIT $L OFFSET $O.
  3. COUNT(*) query for total.
  4. Return { items, total }.
```

Import `appendEvent` from `./coordinationEvents.js`, `ContextHubError` from `../core/errors.js`,
`getDbPool` from `../db/client.js`.

### T3 — `src/services/intake.ts` — triageIntake (complex, FOR UPDATE)

**File:** `src/services/intake.ts` (continued, same file)

```
triageIntake(intake_id, route: TriageRoute):
  1. const client = await pool.connect(); try { BEGIN
  2. SELECT intake_items WHERE intake_id=$1 FOR UPDATE → NOT_FOUND or INTAKE_ALREADY_TRIAGED (status!='received')
  3. If route.route_kind === 'dispute':
       a. Call openDispute({ topic_id: route.topic_id, subject_ref: route.subject_ref,
            parties: route.parties, procedure: route.procedure,
            submitted_by: route.submitted_by, kind: route.kind, weight: route.weight })
            → { dispute, resolution_request_id }.  (openDispute uses its own pool.connect internally)
       b. routed_to = dispute.dispute_id.
  4. Else: routed_to = route.routed_to.
  5. UPDATE intake_items SET status='triaged', topic_id=$2, routed_to=$3 WHERE intake_id=$1 RETURNING *.
  6. COMMIT } catch(e) { ROLLBACK; throw } finally { client.release() }
  7. appendEvent(pool, { topic_id: route.topic_id, actor_id: route.actor_id,
       type:'intake.triaged', subject_type:'intake', subject_id: intake_id,
       payload:{ route_kind: route.route_kind, routed_to } }).
  8. Return TriageResult (+ dispute_id + resolution_request_id if dispute route).
```

Note: `openDispute` is called WITHIN the transaction window but uses its own pool connection.
The `FOR UPDATE` on intake_items ensures no concurrent triage proceeds past step 2.

### T5 — `src/services/disputes.ts`

**File:** `src/services/disputes.ts`

```
openDispute({ topic_id, subject_ref, parties, procedure, submitted_by, kind?, weight? }):
  1. Validate: parties.length >= 2 (BAD_REQUEST); procedure ∈ {'unilateral','collective'}.
  2. pool.query SELECT status FROM topics WHERE topic_id=$1 → NOT_FOUND or TOPIC_NOT_ACTIVE.
  3. pool.query INSERT INTO disputes (topic_id, subject_ref, parties, status) VALUES ... RETURNING *.
  4. Call submitRequest({ topic_id, subject_type:'dispute', subject_id: dispute_id,
       procedure, submitted_by, kind: kind ?? 'dispute_resolution', weight: weight ?? 1 })
       → { request_id, ... }.
  5. pool.query UPDATE disputes SET resolution_request_id=$1 WHERE dispute_id=$2 RETURNING *.
  6. appendEvent(pool, { topic_id, actor_id: submitted_by, type:'dispute.opened',
       subject_type:'dispute', subject_id: dispute_id,
       payload:{ subject_ref, parties, procedure, resolution_request_id: request_id } }).
  7. Return { dispute: updatedRow, resolution_request_id: request_id }.

resolveDispute(dispute_id):
  1. const client = await pool.connect(); try { BEGIN
  2. SELECT * FROM disputes WHERE dispute_id=$1 FOR UPDATE
     → NOT_FOUND; if status='resolved' → ALREADY_RESOLVED.
  3. SELECT status FROM requests WHERE request_id=$1 (the resolution_request_id)
     → if status NOT IN ('approved','returned','rejected') → RESOLUTION_PENDING.
  4. UPDATE disputes SET status='resolved' WHERE dispute_id=$1 RETURNING *.
  5. COMMIT } catch(e) { ROLLBACK; throw } finally { client.release() }
  6. appendEvent(pool, { topic_id: dispute.topic_id, actor_id: submitted_by(from dispute or caller?),
       type:'dispute.resolved', subject_type:'dispute', subject_id: dispute_id,
       payload:{ resolution_request_id, request_status: requestStatus } }).
     → actor_id = dispute.parties[0] (the first party) as the resolution actor.
  7. Return updated dispute row.

getDispute(dispute_id):
  1. SELECT * FROM disputes WHERE dispute_id=$1 → NOT_FOUND.
  2. SELECT requests + request_steps WHERE request_id=resolution_request_id.
  3. Return { ...dispute, resolution_request: { request_id, status, steps } }.

listDisputes(topic_id, opts?):
  1. SELECT * FROM disputes WHERE topic_id=$1 [AND status=$2] ORDER BY created_at DESC LIMIT $L OFFSET $O.
  2. COUNT(*).
  3. Return { disputes, total }.
```

Import: `submitRequest` from `./requests.js`, `appendEvent` from `./coordinationEvents.js`.
