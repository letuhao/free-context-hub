## Sprint scope — what changes vs the Phase 16 DESIGN

The DESIGN was written assuming `ragas==0.2.x` with direct OpenAI client. The actual current stable is `ragas==0.4.3` and the API uses `llm_factory` + `litellm` for OpenAI-compatible local endpoints. **This sprint pins `ragas==0.4.3`** and updates the implementation accordingly. The HTTP API contract (POST /score, GET /health) is unchanged; only the internal wiring shifts.

## Decisions locked at sprint start

| # | Decision | Rationale |
|---|---|---|
| S2-D1 | Pin `ragas==0.4.3` | Latest stable as of 2026-05-24 |
| S2-D2 | Use `litellm` provider (per ragas 0.4.x docs) to point at LM Studio | OpenAI-compatible; matches `JUDGE_AGENT_BASE_URL` env shape |
| S2-D3 | Sidecar reaches LM Studio via `host.docker.internal:1234` | LM Studio runs on host, not in compose network |
| S2-D4 | Embeddings endpoint also via LM Studio (`EMBEDDINGS_BASE_URL` already wired in project) | One model server, two roles (chat + embeddings) |
| S2-D5 | `refusal_correctness` is a CUSTOM Python metric (not ragas-builtin) | Implemented inline in `main.py`; doesn't need ragas extension API |
| S2-D6 | Health endpoint does NOT call LM Studio on every check — only on startup + periodic probe | Avoid spamming the LLM with health pings; ping every 5 min instead |
| S2-D7 | Container exposes port `3005` on host | Free port; documented in DESIGN §4.1 |
| S2-D8 | TS client lives at `src/qc/judge.ts` per DESIGN §8 file layout | No env var yet (will add `RAGAS_JUDGE_URL` to env in 16.3 when runBaseline.ts integration lands) |

## Risks specific to Sprint 16.2

| Risk | Mitigation |
|---|---|
| LM Studio at `host.docker.internal:1234` fails from container | C3 live smoke catches it early; alternative: bind-mount LM Studio's port or use `network_mode: host` |
| Ragas 0.4.x API differs from what we coded | Build incrementally: A4 (skeleton+health) → B1 (route shape) → B3 (real ragas call); if API differs, adjust before B4 |
| Gemma-4-26B-A4B doesn't follow ragas's structured-output prompt reliably | Mitigate by capping `temperature=0` and `seed=42`. If still flaky, add `--num-retries=2` to ragas evaluate calls. Document for follow-up if persistent. |
| Determinism issues per Phase 16 risk R1 (MoE may not be byte-identical) | Sprint 16.2 also includes a 5× determinism probe — run same input 5× and report stddev of each metric. If high, document; Sprint 16.3 may need to pin generated answers in dataset itself. |
| Container build slow due to ragas dependencies | First build ~3-5 min (large dep tree); subsequent builds layer-cache; acceptable for dev. |
