## Group C — `src/services/motions.ts` (the motion lifecycle; topic-event-emitting)

### T5 — `proposeMotion` + module types + constants
- **Files:** `src/services/motions.ts` (new), `src/services/motions.test.ts` (new).
- **Intent:** `MOTION_DEADLINE_DEFAULT_MINUTES=1440` / `_MIN=5` / `_MAX=43200`; types
  (`MotionRecord`, `ProposeResult`, …); `proposeMotion` per §3.1 — validation incl.
  `subject_ref` ≤ 256 and `deadline_minutes` an integer in `[MIN,MAX]` (default 1440);
  pre-`BEGIN` reads (topic → `NOT_FOUND`/`topic_closed`; body — none **or**
  `body.project_id ≠ topic.project_id` → `body_not_found`; participant → `not_participant`);
  txn: `INSERT motions` (`status='proposed'`) + `appendEvent('motion.proposed')`.
- **TDD:** valid → `{proposed, motion_id}` + a `motion.proposed` event; non-participant →
  `not_participant`; unknown body → `body_not_found`; **cross-project body → `body_not_found`**;
  closed topic → `topic_closed`; `deadline_minutes` out of `[5,43200]` → `BAD_REQUEST`.
- **Verify:** `npx tsx --test src/services/motions.test.ts`.

### T6 — `secondMotion`
- **Files:** `src/services/motions.ts`, `…test.ts`.
- **Intent:** §3.2 — txn, lock `motions … FOR UPDATE`; `not_found` / `conflict` (status ≠
  `proposed`); plain-read topic → `topic_closed`; `actor = proposed_by` →
  `self_second_forbidden`; `body_members` check → `not_member`; `UPDATE motions SET
  status='balloting', seconded_by=$` + `appendEvent('motion.seconded')`.
- **TDD:** member ≠ proposer → motion `balloting` + `motion.seconded`; the proposer →
  `self_second_forbidden`; a non-member → `not_member`; a non-`proposed` motion → `conflict`;
  a closed topic → `topic_closed`.
- **Verify:** per-file test.

### T7 — `castVote`
- **Files:** `src/services/motions.ts`, `…test.ts`.
- **Intent:** §3.3 — `proxy_for == actor_id` → `BAD_REQUEST`; txn, lock `motions … FOR UPDATE`;
  `not_found` / `not_balloting` (status ≠ `balloting`) / `balloting_closed` (`now() ≥
  deadline`); plain-read topic → `topic_closed`; `body_members` (the **principal**) →
  `not_member` + capture `vote_weight`; `INSERT votes … ON CONFLICT DO NOTHING` (weight
  snapshotted) → 0 rows → `already_voted`; `appendEvent('vote.cast', actor_id = proxy_for ??
  actor_id, payload incl. principal)`.
- **TDD:** member during `balloting` → `vote_recorded` + `votes.weight == body_members
  .vote_weight`; non-member → `not_member`; non-`balloting` → `not_balloting`; past `deadline`
  → `balloting_closed`; re-cast → `already_voted`; a proxy ballot → row keyed on the principal
  with `proxy_for` set; `proxy_for == actor_id` → `BAD_REQUEST`.
- **Verify:** per-file test.

### T8 — the §4 tally query + `tallyMotion`
- **Files:** `src/services/motions.ts`, `…test.ts`.
- **Intent:** the DESIGN §4 aggregate-and-decide SQL (one statement, Postgres `NUMERIC` —
  `f`/`a`/`ab` via `FILTER`, `quorum_met`, `carried` = `(f+a) > 0 AND f >= $threshold*(f+a)`);
  `tallyMotion` per §3.5 — txn, lock `motions … FOR UPDATE`; `not_found` / `not_balloting` /
  **`now() < deadline → balloting_open`** (BLOCK-1); topic → `topic_closed`; read body's
  `quorum`/`threshold`; run the §4 query; map `quorum_met`/`carried` → `lapsed`/`failed`/
  `carried`; `UPDATE motions SET status=outcome, tally=tally_json`; `appendEvent('motion
  .tallied', actor_id='system:tally')`.
- **TDD (the tests set `deadline` into the past via direct SQL — the 15.3 escalation-test
  precedent):** quorum unmet → `lapsed`; quorum + threshold met → `carried`; quorum met,
  threshold unmet → `failed`; **all-abstain → `failed`** (no divide-by-zero); **tie at
  `threshold=0.5` → `carried`** (inclusive threshold); **pre-`deadline` tally →
  `balloting_open`**; non-`balloting` → `not_balloting`; **weight-snapshot:** change a member's
  `vote_weight` after they vote → the tally uses the cast-time weight.
- **Verify:** per-file test.

### T9 — `vetoMotion`
- **Files:** `src/services/motions.ts`, `…test.ts`.
- **Intent:** §3.4 — txn, lock `motions … FOR UPDATE`; `not_found` / `not_balloting`; topic →
  `topic_closed`; read `decision_bodies.veto_holders`, `actor ∉ veto_holders` →
  `not_veto_holder`; `UPDATE motions SET status='vetoed'` + `appendEvent('motion.vetoed')`.
- **TDD:** a `veto_holders` member while `balloting` → `vetoed` + `motion.vetoed`; a non-holder
  → `not_veto_holder`; a non-`balloting` motion → `not_balloting`; **mutual exclusion** — a
  `vetoed` motion then `tallyMotion` → `not_balloting`; a `carried`/`failed` motion then
  `vetoMotion` → `not_balloting`.
- **Verify:** per-file test.

### T10 — `getMotion` + `listMotions`
- **Files:** `src/services/motions.ts`, `…test.ts`.
- **Intent:** §3.6 — `getMotion` one query, motion + `json_agg(votes)`; `null` on no row.

- **TDD:** `getMotion` returns motion + votes; unknown id → `null`; `listMotions` unknown

- **Verify:** per-file test.

## Group D — the sweep

### T11 — `sweepExpiredMotions` + the third-sweep scheduler wiring
- **Files:** `src/services/coordinationSweep.ts` (mod), `src/services/coordinationSweep.test.ts`
  (mod).
- **Intent:** §5 — `ExpiredMotionsSweepResult`; `sweepExpiredMotions` (scan `status IN
  ('proposed','balloting') AND deadline < now()-grace`; per-motion §0.1-loop txn; lock
  `motion → topics` both `FOR UPDATE`; closed-topic → skip; `proposed` → `lapsed` +
  `appendEvent('motion.tallied', outcome:'lapsed', reason:'not_seconded')`; `balloting` → the
  §4 query → `UPDATE` + `appendEvent('motion.tallied', actor_id='system:sweep', auto:true)`);
  add `_sweepExpiredMotions` to the test-injectable deps + `__set/__reset…ForTest`; the
  scheduler `cycle` calls it third, in the one advisory-lock hold.
- **TDD:** a `balloting` motion past `deadline` → auto-tallied to the correct outcome; a
  `proposed` motion past `deadline` → `lapsed`; a motion on a **closed** topic → skipped (no
  mutation/event); one bad + one good expired motion → the good one still resolves (crash
  isolation — the 15.3 T20 pattern).
- **Verify:** `npx tsx --test src/services/coordinationSweep.test.ts`.
