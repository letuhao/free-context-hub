## Sprint scope

This sprint connects the dataset (16.1) + judge (16.2) into a single end-to-end gen-eval pipeline. Running `npm run qc:baseline:gen --tag <t>` will:
1. Load each surface's golden set
2. For each query: run retrieval → if `ideal_answer` is set, synthesize an answer using a pinned prompt + LLM → POST to ragas-judge → attach scores
3. Emit baseline JSON schema v1.1 with new `generation` block + `judge` provenance
4. Render scorecard MD with gen-metric tables
5. Detect regressions vs prior tag via `diffBaselines.ts`

## Decisions locked at sprint start

| # | Decision | Rationale |
|---|---|---|
| S3-D1 | Reuse Gemma-4-26B-A4B as BOTH answerer + judge (single model in LM Studio) | Same hardware constraint; separate prompts give them different roles. Tradeoff documented: same-model bias may inflate scores. Phase 17 may swap answerer to a smaller/faster model. |
| S3-D2 | Synthesizer prompt templates live in `src/qc/templates/synthesizer.<surface>.txt` | Per DESIGN §5.3; hashed and recorded in baseline manifest for drift detection |
| S3-D3 | Redis cache is OPTIONAL for 16.3 first pass | First baseline run with ~152 rows × 4-5 metrics each = ~700 judge calls × ~10s each = ~2 hours. Acceptable for nightly. Add cache in 16.4 if needed. |
| S3-D4 | diff extension is minimal (full-surface gen aggregates only) | Per-category drift detection adds complexity; ship MVP first, refine in 16.4 |
| S3-D5 | Default `--gen-eval=auto`: runs gen when row has `ideal_answer`, skips otherwise. `--gen-eval=off` for retrieval-only. | Backwards-compatible with existing `npm run qc:baseline` callers |
| S3-D6 | Threshold gating: WARN-only per Phase 16 D5 | No PR-blocking yet; 2 weeks of baseline data first |
| S3-D7 | 5x determinism probe deferred to 16.4 | Pre-baseline issue 16.2-OPEN-1 (context_recall noise). Quantification needs a separate `--determinism-probe` flag; not blocking 16.3 ship. |

## Risks specific to Sprint 16.3

| Risk | Mitigation |
|---|---|
| Same-model bias (answerer=judge=Gemma 4) inflates faithfulness scores | Document in readiness; Phase 17 candidate: swap answerer to a different model for adversarial measurement |
| Context fetch per query takes long (some surfaces need MCP/DB round-trip per hit) | Cache fetched content per `(surface, key)` in-memory across a single baseline run |
| Smoke run takes >30 min wall-clock and user wants to checkpoint mid-run | Add `--max-rows N` flag to limit per-surface row count (smoke = 3-5; full = unlimited) |
| Judge sidecar fails on a row → blocks baseline | Per-row error: catch, log, set `generation.error`, continue. Don't kill the whole baseline. |
| LM Studio auto-unloads mid-baseline (16.2-OPEN-2) | First call latency is bounded; if a chain of calls fails with `No models loaded`, abort with clear error and instruct user. Don't retry indefinitely. |
