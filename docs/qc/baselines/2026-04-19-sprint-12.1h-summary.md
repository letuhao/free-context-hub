## Recommendation by use case

- **Production (user-facing quality matters):** `RERANK_TYPE=generative` (default, unchanged). Best quality; non-determinism is not a user-visible problem.
- **QC measurement (determinism matters):** `RERANK_TYPE=cross-encoder` + `RERANK_MODEL=gte-reranker-modernbert-base`. Deterministic + better quality than alternatives.
- **Fast QC iteration (speed matters more than top-K precision):** `RERANK_TYPE=api` (now with TEI+minilm). 18× faster than bge, near-identical quality, fully deterministic. Good for rapid iteration when MRR/recall@10 are the primary metrics.

## What we learned across 12.1e4 → 12.1h

This 4-sprint arc investigated rerank behavior exhaustively:
- **12.1e4:** LLM generative rerank non-deterministic across sessions (~0.027 MRR drift per 40q baseline).
- **12.1f:** LM Studio `/v1/embeddings` + cosine works ONLY for bi-encoder-compatible models (gte); bge/jina fail (near-random output).
- **12.1g:** External TEI `/rerank` endpoint unlocks true cross-encoders, but bge-v2-m3 specifically underperforms.
- **12.1h:** Tried jina (architecturally incompatible with TEI) + minilm (functional, fast, same quality as bge).

**Closing finding:** Generative LLM rerank gives ~0.05-0.07 MRR boost over any cross-encoder we've tested. That boost appears to be genuine LLM reasoning (semantic equivalence, indirect references) that ranker-sized models don't replicate. No cross-encoder in our tested set beats it.
