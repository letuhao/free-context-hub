# α × HL grid sweep summary — Sprint 12.1e4

## Recommendation (one line)

**Keep `LESSONS_SALIENCE_ALPHA=0.10` and `LESSONS_SALIENCE_HALF_LIFE_DAYS=7`** — α has ZERO effect on this goldenset under NOWRITE (confirmed via rerank-off validation: α=0.05 and α=0.10 produced byte-identical metrics across 40 queries). The apparent α-sensitivity in the initial 8-run grid (original recommendation below) was **entirely LLM rerank drift across container recreates**, not an α signal. Under rerank-off, all configs are fully deterministic and α-independent.

## ⚠️ CORRECTION (investigation at POST-REVIEW)

The original 8-run α × HL grid (initial summary below) showed apparent α-driven outliers at (HL=7, α=0.10) and (HL=30, α=0.20). POST-REVIEW option-3 investigation (code trace + 3 rerank-off validation runs) revealed:

1. **Salience blend is a no-op for this goldenset.** Zero of the 60 candidate lessons per query have any `lesson_access_log` entries (including bootstrap — guardrail_audit_logs.rule_id doesn't match any current guardrail lesson_id). `blendHybridScore` returns hybridScore unchanged when salience=undefined. So α literally multiplies zero.

2. **The "outlier" pattern was LLM rerank drift.** A same-config rerun (HL=7 α=0.05 repeated ~90min later) produced 7/40 queries with different `found_ranks` including the sprint-11-closeout MISS. Repeating the same config DOES NOT reproduce — the reranker's output shifts across container recreates (likely LM Studio state drift at temp=0).

3. **Under rerank-off (`DISTILLATION_ENABLED=false`):** same config is fully deterministic across recreates. α=0.05 and α=0.10 produce 0/40 query differences. The metric floor without rerank is MRR=0.9198, nDCG@10=0.9100 — rerank genuinely improves quality by ~0.06 MRR when it "works" but non-deterministically.

## Why α is a true no-op on current state

- Zero of 60 candidate lessons have access-log entries → salienceMap is empty → `blendHybridScore(hybrid, undefined, α, rel) === hybrid` regardless of α value.
- This means α is not "maybe small effect, below noise" — it's **exactly zero effect** on the current corpus × access-log state.
- Mathematical fact, not empirical: the code short-circuits before the `(1 + α × sal × rel)` multiplier ever runs.

## Outlier analysis

Both "⚠️" runs (HL=7 α=0.10, HL=30 α=0.20) dropped cross-topic nDCG@10 from 0.8467 → 0.5945. Investigating which query missed:

The two affected runs are the ONLY ones where a single cross-topic query landed at rank > 10 instead of within. At 40 queries × 1 sample, retrieval ordering depends on: semantic score (from embedding service), FTS score (from postgres), dedup, rerank. None of these are parameterized by α. So two runs at different (HL, α) combos producing the SAME borderline-miss pattern suggests this is **retrieval-service jitter** — likely LM Studio embedding service returning very slightly different cosine similarities between container restarts, causing a rank-10-vs-rank-11 borderline to flip on a single query.

**This is not a bug.** It's the same `measurement-jitter` friction class documented since Sprint 12.0 — within-session runs are deterministic, but cross-container-restart runs can have microscopic embedding drift. The 8 α × HL runs here are all cross-restart; each baseline gets its own container recreate + embedding-cache warmup.

## Notable per-query shifts

Outside the two outliers, **zero queries changed their found_ranks across the 8 runs.** Every non-outlier run produced an identical ranking. This is extreme uniformity — α × HL has no effect on 40/40 queries in the non-outlier runs.

## Decision-support

1. **α=0.10 is fine.** 6 of 8 cells (including the current default HL=7, α=0.10 — adjusted for the outlier) cluster tightly at MRR=0.9784 / nDCG@10 in [0.9630, 0.9717]. α choice within the tested range is **dominated by measurement jitter**.

2. **HL=30 α=0.05 gets the HIGHEST aggregate nDCG@10 (0.9717)**, beating HL=7 α=0.10 (0.9630) by +0.0087. But this is within noise floor (0.0134). Not strong enough signal to warrant a default change.

3. **Higher α at HL=30 doesn't hurt** (α=0.50 nDCG@10=0.9676). Earlier theory that "HL=30 + high α over-boosts guardrails" isn't supported by this grid — the α=0.20 outlier is jitter, not systematic over-boost.

4. **Why α has no effect.** At HL=7 NOWRITE, bootstrap-salience is ~10⁻⁴ per row; even α=0.50 gives boost ~ 1.00004 (negligible). At HL=30 NOWRITE, bootstrap-salience saturates at ~1.0 for guardrail lessons; α=0.50 gives boost up to ~1.35 — but ONLY for guardrail lessons, which the 40q goldenset barely exercises outside the already-saturated duplicate-trap group. The feature has no room to move rankings on this goldenset.

5. **Implication for the salience feature.** Under NOWRITE (clean measurement), the salience blend contributes no measurable improvement over pure hybrid retrieval. This is consistent with Sprint 12.1e3's finding that HL tuning is also no-op under NOWRITE. Production behavior (with accumulating real user traffic) will differ, but this goldenset can't measure that state.

## What this rules in / rules out

- **Rules out:** α tuning is a meaningful lever on the current goldenset. Default 0.10 is defensible; no α value clearly beats it.
- **Rules in (candidate):** The goldenset is too bootstrap-limited to measure salience. A future sprint could seed realistic consideration-search + consumption traffic (not goldenset-pollution style but simulated user distribution) to create measurable salience signal.
- **Confirms:** NOWRITE measurement is still subject to rank-10-borderline jitter. For any metric where a single query's rank-10→11 flip drops group nDCG@10 by 0.25, samples=1 isn't robust. Future salience-sensitive sprints should use samples=3+ OR ignore runs with known-jitter-borderline behavior.

## Implications for prior Phase 12 sprints

**12.1e2 "HL=30 wins" finding:** was partially within-run drift (addressed in 12.1e3) AND partially LLM rerank cross-session drift (NEW finding). Combined, almost all of 12.1e2's apparent signal was measurement artifact.

**12.1e3 "HL=7 wins after clean-state revert":** same caveat — the 2×2 was run at different times with rerank ON. Any "HL=7 ≥ HL=30 on clean state" conclusion is subject to the same drift.

**Good news:** the HL default revert to 7 in 12.1e3 is STILL CORRECT because the data doesn't support HL=30, and absent clean evidence, restoring the original deliberate choice (7, per 12.1c design intent) is the right call. But our confidence in "HL=30 doesn't help" should be tempered.

## Recommendation (expanded)

**No changes to `src/env.ts` defaults.** Keep α=0.10, HL=7. α is literally a no-op multiplier on current state (blend short-circuits). HL decision stands from 12.1e3.

**Measurement methodology changes for future sprints:**
1. **Default `DISTILLATION_ENABLED=false` for QC baseline runs.** Rerank adds ~100× runtime and cross-session drift that exceeds most parameter signals. Measure without rerank; document separately if rerank's quality contribution is the target.
2. **If rerank must be measured, run A/B matrices in a single session back-to-back** (minimize LM Studio state drift window).
3. **Expose `DISTILLATION_ENABLED` override in docker-compose.yml** (shipped this sprint as part of the investigation: `${DISTILLATION_ENABLED:-true}` in mcp env block).

**Access log distribution:** bootstrap-only state under-represents production. Future salience measurement should seed realistic consideration-search + consumption distribution to exercise the feature.

## Friction classes added this sprint

1. **`llm-rerank-cross-session-drift`** — LLM reranker at temp=0 produces non-deterministic output across container recreates / time-separated sessions, likely due to LM Studio internal state drift. Single-container same-session calls are deterministic; cross-session are not. Drift magnitude: ~0.027 MRR / ~0.031 nDCG@10 observed, often producing MISS/HIT flips at rank-10 borderline.
2. **`salience-blend-noop-when-no-access-history`** — When candidate lessons have no `lesson_access_log` entries (common for fresh-corpus queries or queries over lessons that haven't been surfaced/consumed), `blendHybridScore` short-circuits to hybridScore unchanged. α has exactly zero effect. Observable via `explanations` field showing "salience: no access history for any candidate (N lessons)".
