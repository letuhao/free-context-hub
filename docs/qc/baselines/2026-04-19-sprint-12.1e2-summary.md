**Key aggregate observation.** HL=3, HL=7, and HL=14 are **statistically identical** (all within noise floor). HL=30 is the first value above noise floor on MRR (+0.0254, above floor) and nDCG@10 (+0.0405, 3× floor).

**Mechanistic explanation.** 6849 of 6939 access-log rows are prior-baseline consideration-search writes < 1 day old. For those, `exp(-t/HL × ln2)` is ~1.0 at any half-life ≥ 3 days. Only the 90 audit-bootstrap rows (90 days old, seeded from `guardrail_audit_logs.created_at`) differ meaningfully across half-lives. At HL=3, bootstrap weight is ~10⁻⁹ (effectively zero). At HL=30, bootstrap weight is ~0.12 — finally enough to register in ranking.

## Clean-state A/B (post-review investigation)

**Procedure.** Truncated all `context='consideration-search'` rows (6849 rows) — keeping only the 90 audit-bootstrap rows. Re-ran HL=7 and HL=30 at `--samples 1` (retrieval is deterministic; samples count only affects latency).

**Results:**

|---|---:|---:|---:|---:|
| HL=7 polluted (in sweep) | 0.9730 | 0.9514 | 0.9380 | 0.9330 |
| HL=30 polluted (in sweep) | 1.0000 | 0.9768 | 0.9697 | 0.9735 |
| **HL=7 CLEAN** | **1.0000** | **0.9581** | **0.9525** | **0.9495** |
| **HL=30 CLEAN** | **1.0000** | **0.9865** | **0.9691** | **0.9649** |

**Clean-state delta (HL=30 vs HL=7):** MRR **+0.0284** (above noise floor 0.020), nDCG@10 **+0.0154** (above noise floor 0.013). **Signal survives pollution removal — it's real.**

**The polluted-run paraphrase regression DISAPPEARS on clean state.** Undici query at HL=30 clean hits target at **rank-2** (not rank-7 like polluted run). The only queries still outranking undici are `a6a3a11d` (e2e dedup-wiring) and `f142bda7` (retrieval-layer changes) — both with hybrid scores close to the undici lesson but without the audit-bootstrap boost.

**Interpretation — goldenset-pollution friction class.** The 6849 consideration-search rows in lesson_access_log were accumulated from 20+ baseline runs across 12.1c-12.1e1. The goldenset's duplicate-trap queries target fixture lessons (`impexp-*`, `agent-bootstrap-e2e-*`, etc.) with 9-20 targets each; each baseline run wrote 9-20 consideration-search rows for each fixture lesson. Over 20+ runs, fixture lessons accumulated hundreds of rows → high salience → inflated ranks at HL=30 → displaced legitimate targets for abstract queries. This is a measurement methodology artifact, not a production concern (baseline runs are dev-only).

**Mitigation for future sprints.** Before measuring salience-sensitive behavior, truncate `consideration-search` rows or use `lesson_access_log_backup_*` tables to isolate bootstrap-only state. Or introduce a `--no-write` flag to `qc:baseline` that reads salience but doesn't accumulate new writes. Filed as candidate for 12.1e2 follow-up.

**Post-investigation access-log state:** 90 rows (audit-bootstrap only). Pre-sprint pollution deleted. Backup preserved in `lesson_access_log_backup_20260419` (6939 rows) in case we want to restore later.

## Decision-support

1. **The aggregate nDCG@10 win (+0.0405) is 3× noise floor** — beyond doubt this is real signal, not jitter.
2. **Cross-topic group gain (+0.358) is the biggest single contributor** — two previously-undetectable queries now hit. This alone motivates HL=30.
3. **The paraphrase regression is a single query** — `lesson-para-undici-node-mismatch` drops rank-1 → rank-7. One data point doesn't warrant keeping 7d; the win on 2+ cross-topic queries + 14+ ambig queries dominates.
4. **Caveat — multi-target "tightening" has a cost.** Ambig queries drop some targets out of top-10 under HL=30 (e.g., measurement-methodology [1,3,5,8] → [1,2]). nDCG@10 still improves because the best-ranked target gets more credit. But if a future use case needs BROAD retrieval coverage (multiple relevant docs surfaced to user), HL=30's tightening may be the wrong trade. For THIS project's use case (single best answer for agent retrieval), tightening is a feature.
5. **The null result at HL=3/7/14 is itself a finding.** Half-life < 14 days on this access-log state is effectively a no-op — salience provides no ranking benefit below HL=30 because bootstrap contributions decay to zero. Users considering salience tuning should know: the knob only moves retrieval when half-life is long enough to preserve old audit-bootstrap / consumption signal.

## Recommendation (expanded)

**Update `src/env.ts` LESSONS_SALIENCE_HALF_LIFE_DAYS default from 7 to 30.**

- Consider a follow-up query-specific guard: for `semantic-paraphrase`-style queries where no bootstrap signal exists for the target, salience boost may harm ranking. A future improvement: detect low-salience-target queries and reduce α for them. That's 12.2 / later scope — out of 12.1e2.
