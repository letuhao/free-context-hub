## CLI surface (final)

```
npx tsx src/qc/runBaseline.ts                       # default: gen-eval=auto
npm run qc:baseline                                  # alias of above
npm run qc:baseline:retrieval                        # gen-eval OFF (pre-16.3 behavior)
npm run qc:baseline:gen                              # gen-eval ON (strict)

Flags:
  --tag <name>             # archive tag (required for non-temp runs)
  --surfaces lessons,code,chunks,global  # filter (default: all)
  --gen-eval auto|on|off   # default auto: runs when row has ideal_answer
  --judge-url <url>        # default RAGAS_JUDGE_URL env or http://localhost:3005
  --top-k-contexts N       # default 5 — how many retrieval hits feed the synth
  --max-rows N             # smoke runs; cap rows per surface
  --samples N              # latency-sample count (default 3)
  --k N                    # retrieval top-K (default 10)
  --control                # back-to-back noise-floor mode
```

## Same-model bias

By default the **answerer = judge = Gemma 4** (or whatever LM Studio has loaded). This creates a small but real bias: a judge that's the same model as the answerer may be more lenient on artifacts the model "likes." Phase 17 candidate: swap answerer to a smaller / faster model (e.g. a 4B-12B instruct) while keeping the judge at 26B+. Estimated bias impact: 3-5pp on faithfulness based on published reports; quantify with a paired-model probe in 16.4.

## What's NOT in this sprint (intentionally — Phase 17 candidates)

- **Redis cache for judge results** — synth+judge per row ~10-30s; cache becomes valuable when re-running for retrieval A/B. Phase 17.x.
- **Per-category drift detection in diff** — current diff is full-surface aggregates only. By-category (multi_hop / no_answer / etc.) would catch hidden regressions.
- **5x determinism probe** — quantify 16.2-OPEN-1 (context_recall noise). Will inform whether to pin generated answers in dataset.
- **Production-fidelity Path B** — call the actual chat sidebar instead of the controlled synthesizer. Real-traffic eval.
- **CI integration / PR gating** — per Phase 16 D5, WARN-mode for 2 weeks of nightly data first.
- **Advanced retrieval techniques** — RRF, HyDE, semantic chunking, query rewrite, reranker tuning. Each A/B'd against the new baseline. **This is Phase 17 proper.**

## Persistence-worthy lessons (RETRO candidates)

| Topic | Why worth persisting |
|---|---|
| Schema-bump pattern: extend types optionally → bump schema_version → handle missing-field gracefully in downstream diff | Generic pattern; will reapply when we add per-category drift, embedding metrics, etc. |
