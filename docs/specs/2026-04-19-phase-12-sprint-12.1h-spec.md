# Spec — Phase 12 Sprint 12.1h — Alternative reranker models via TEI

Follow-up to Sprint 12.1g. TEI infrastructure is deployed; 12.1g's choice (`bge-reranker-v2-m3`) underperformed. This sprint swaps models to see if a different reranker wins.

## Scope

**IN SCOPE:**
1. **Primary: swap tei-rerank to `jinaai/jina-reranker-v2-base-multilingual`.** Edit `docker-compose.yml` `--model-id`, recreate TEI, wait for model download + ready.
2. **Baseline run** with `RERANK_TYPE=api`, NO_WRITE=true, α=0.10, HL=7, samples=1.
3. **Decision gate:**
   - If jina MRR ≥ 0.98 AND deterministic → **STOP. Winner. Update default.**
   - If jina clearly loses (MRR < gte's 0.9538) → **STOP. Try fallback.**
   - If jina is ambiguous (between no-rerank and gte) → proceed to fallback.
4. **Fallback: swap tei-rerank to `Qwen/Qwen3-Reranker-0.6B`.** Same procedure.
5. **Summary doc** with full 6+ way comparison (prior 5 + jina + optional qwen).
6. **Determinism check** on best-performing new model (same-config repeat, 0/40 diffs required).
7. **If a winner exists:** update `src/env.ts` `RERANK_TYPE` default to `api`. Update `docker-compose.yml` `tei-rerank` `--model-id` to the winning model permanently.

**OUT OF SCOPE:**
- Trying more than 2 models (12.1h is explicitly capped; if both fail, rerank loop closes).
- Code path changes (12.1g's `rerankExternalApi` already works).
- Salience math changes.

## Acceptance criteria

A1. `docker-compose.yml` `tei-rerank` service has one of: jina-reranker-v2-base-multilingual OR Qwen3-Reranker-0.6B as active model (depending on winner).
A2. At least 1 new baseline archive at `docs/qc/baselines/2026-04-19-sprint-12.1h-{model-name}.{json,md}` (possibly 2 if fallback ran).
A3. `lesson_access_log` count stays at 90 throughout.
A4. Summary doc `docs/qc/baselines/2026-04-19-sprint-12.1h-summary.md` with top-line recommendation + 6+-way matrix + per-group + determinism verdict.
A5. SESSION_PATCH entry.
A6. `npx tsc --noEmit` clean (no code changes expected, but confirm).
A7. If winner exists: `src/env.ts` `RERANK_TYPE` default changes to `api`; tests still pass; docker-compose.yml `--model-id` updated to winner.

## Decisions locked

**D1 — Primary model:** `jinaai/jina-reranker-v2-base-multilingual` (~280M params, newer architecture, multilingual). HF-hub compatible for TEI.

**D2 — Fallback:** `Qwen/Qwen3-Reranker-0.6B` (smallest qwen reranker, ~600M params, English-leaning training).

**D3 — Winner threshold:** MRR ≥ 0.98 AND nDCG@10 ≥ 0.96 AND deterministic. Matches 12.1g's design §9 "strong win" criterion.

**D4 — Cap at 2 models.** If both lose, rerank optimization loop closes. No escalation to 3rd model.

**D5 — No same-session back-to-back comparison.** Since TEI is deterministic by 12.1g finding, same-session repeat is cheap but not essential unless quality is strong enough to warrant it.

## Risks + mitigations

**R1 — Model not available on HF Hub / incompatible with TEI.** Mitigation: TEI's error message is clear on startup; document fallback + move on.

**R2 — Model download takes 5-10 min.** One-time per model swap. Mitigation: document expected wait; use start_period=300s healthcheck (already in place).

**R3 — Neither model wins.** Expected plausible outcome. Mitigation: sprint ships honest null; rerank optimization loop closes.

**R4 — TEI model cache grows large.** jina + qwen + existing bge = ~3 models × ~300MB = ~1GB. Named volume persists; user can prune later. Acceptable.
