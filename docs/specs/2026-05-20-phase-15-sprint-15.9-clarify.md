## Risks

1. **MCP SDK discriminated-union issue (DEFERRED-007)**: if I declare chain as
   a `z.discriminatedUnion('kind', ...)`, `tool/call` returns `_zod` error. Must
   use a flat optional shape (`kind: z.string()` + all sub-fields optional).
   Acceptable trade-off (schema is looser but no SDK breakage).

2. **Test fragility**: adding event-ordering assertions to existing 15.6 tests
   risks false positives if the appendEvent serial allocation has any non-
   monotonic edge. Verified by reading `coordinationEvents.ts` ‚Äî appendEvent
   uses `UPDATE topics SET next_seq = next_seq + 1 RETURNING next_seq` (atomic
   monotonic per-topic seq). Ordering by `seq` is reliable.

3. **No code changes**: this is a docs/test/schema-only sprint. Probability of
   regressing 648/648 is low. Defensive baseline: re-run full `npm test` after
   each task.
