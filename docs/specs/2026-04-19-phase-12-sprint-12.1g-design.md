# Design — Phase 12 Sprint 12.1g

## 1. docker-compose.yml `tei-rerank` service block

Placement: as a sibling service alongside `db`, `neo4j`, `mcp` etc.

```yaml
  # Phase 12 Sprint 12.1g — HuggingFace text-embeddings-inference for
  # true cross-encoder rerank. Supports /rerank endpoint with (query, texts)
  # pair scoring — unblocks bge-reranker-v2-m3, jina-reranker-v3, and other
  # cross-encoders that return garbage via LM Studio's /v1/embeddings.
  # CPU-only to avoid VRAM contention with LM Studio.
  tei-rerank:
    image: ghcr.io/huggingface/text-embeddings-inference:cpu-latest
    command: ["--model-id", "BAAI/bge-reranker-v2-m3"]
    volumes:
      - tei_model_cache:/data
    ports:
      - "8080:80"  # dev/debug; mcp reaches via docker network
    healthcheck:
      test: ["CMD-SHELL", "wget --quiet --spider --tries=1 http://localhost:80/health || exit 1"]
      interval: 15s
      timeout: 5s
      retries: 20
      start_period: 300s  # first run downloads ~560MB model — up to 4min
    restart: unless-stopped
```

Add `tei_model_cache` to the top-level `volumes:` block at file bottom.

**Why port 8080 is exposed:** dev can probe the rerank endpoint directly for debugging without hopping through mcp. Not required for production; could remove.

**Why `restart: unless-stopped`:** matches db, neo4j, etc.

**healthcheck `/health`:** TEI exposes it (verified empirically in CLARIFY — `/info` endpoint responds; `/health` is the standard TEI endpoint).

## 2. mcp service updates

Add `depends_on` for tei-rerank + env var defaulting to service-name URL:

```yaml
  mcp:
    # ... existing fields unchanged ...
    depends_on:
      - db
      - neo4j
      - rabbitmq
      - redis
      - tei-rerank     # NEW
    environment:
      # ... existing env ...
      RERANK_TYPE: ${RERANK_TYPE:-generative}
      RERANK_MODEL: ${RERANK_MODEL:-}
      RERANK_BASE_URL: ${RERANK_BASE_URL:-http://tei-rerank:80}   # NEW — default points at the service
```

**Note:** `RERANK_BASE_URL` already existed in the codebase (env.ts). It was optional with no default, meaning when cross-encoder or generative mode was active, it fell back to DISTILLATION_BASE_URL or EMBEDDINGS_BASE_URL. Now we default it to the tei-rerank service URL when unset. Backward-compat: if user sets RERANK_BASE_URL explicitly, their value wins.

But wait — this changes generative + cross-encoder code paths too (they use `rerankBaseUrl()` which reads `RERANK_BASE_URL`). If the default shifts to `http://tei-rerank:80`, those paths would POST /v1/chat/completions to TEI which would fail.

**Fix: don't default `RERANK_BASE_URL` to TEI URL.** Instead, hardcode the TEI URL for `RERANK_TYPE=api` dispatch, OR use a separate env var `TEI_RERANK_URL`. Cleaner: the `rerankExternalApi()` function reads `RERANK_BASE_URL` with fallback to `http://tei-rerank:80` INTERNALLY. Existing generative/cross-encoder paths use `rerankBaseUrl()` as before.

**Revised plan:**
- `RERANK_BASE_URL` env var semantics unchanged (optional override of rerank server URL).
- `rerankExternalApi()` uses `env.RERANK_BASE_URL ?? 'http://tei-rerank:80'` — TEI service URL is the hardcoded fallback for api mode.
- docker-compose.yml does NOT set `RERANK_BASE_URL` default.

## 4. `rerankExternalApi()` signature + behavior

Placement: alongside `rerankGenerative` and `rerankCrossEncoder` in `src/services/lessons.ts`.

```typescript
/**
 * External-API reranker: POSTs query + candidate texts to a Cohere/TEI-compatible
 * /rerank endpoint and returns the new index order sorted by server-side score.
 *
 * Works with: HuggingFace text-embeddings-inference (TEI), Infinity, Cohere.
 * Requires: RERANK_BASE_URL pointing at a server that exposes /rerank with
 * input {query, texts: [...]} and response [{index, score}, ...].
 *
 * Phase 12 Sprint 12.1g — unblocks true cross-encoder rerank models
 * (bge-reranker, jina-reranker, etc.) that can't be used via LM Studio's
 * /v1/embeddings bi-encoder path.
 */
async function rerankExternalApi(query: string, candidates: RerankCandidate[]): Promise<number[]> {
  const env = getEnv();
  const baseUrl = env.RERANK_BASE_URL ?? 'http://tei-rerank:80';

  const ac = new AbortController();
  const timeoutMs = (env.RERANK_TIMEOUT_MS ?? 10000) + 5000;
  const t = setTimeout(() => ac.abort(), timeoutMs);

  try {
    const texts = candidates.map(c => `${c.title}. ${c.snippet}`);
    const res = await fetch(`${baseUrl}/rerank`, {
      method: 'POST',
      headers: rerankHeaders(),  // already handles RERANK_API_KEY if set
      signal: ac.signal,
      body: JSON.stringify({ query, texts }),
    });

    if (!res.ok) {
      logger.warn({ status: res.status }, 'external-api rerank: HTTP error');
      return candidates.map(c => c.index);
    }

    const json = (await res.json()) as Array<{ index: number; score: number }>;
    if (!Array.isArray(json) || json.length === 0) {
      logger.warn({ json }, 'external-api rerank: empty or malformed response');
      return candidates.map(c => c.index);
    }

    // json is sorted by score DESC; map server-side indices back to our input indices.
    const order = json.map(r => candidates[r.index]?.index).filter((v): v is number => v !== undefined);

    logger.info({
      query: query.slice(0, 60),
      candidates: candidates.length,
      top3: order.slice(0, 3),
      top3_scores: json.slice(0, 3).map(r => r.score.toFixed(5)),
      mode: 'external-api',
    }, 'lesson rerank: done');

    return order;
  } catch (err) {
    logger.warn({ error: err instanceof Error ? err.message : String(err) }, 'external-api rerank: failed');
    return candidates.map(c => c.index);
  } finally { clearTimeout(t); }
}
```

**Failure modes:** network error, 5xx from TEI, malformed response. All fall back to `candidates.map(c => c.index)` — original order, effectively no-op. Same pattern as `rerankGenerative` and `rerankCrossEncoder`.

**Note:** `candidates[r.index].index` maps server-side index (0-14 for 15 candidates) back to the CALLER'S index field — which may not match 0-14 if the caller numbered differently. Safe.

## 5. Dispatch update in `rerankLessons`

```typescript
async function rerankLessons(params: {
  query: string;
  candidates: RerankCandidate[];
}): Promise<number[]> {
  const env = getEnv();
  const model = env.RERANK_MODEL ?? env.DISTILLATION_MODEL;

  // 12.1g — new api dispatch. No DISTILLATION_MODEL check here; TEI
  // doesn't use LM Studio at all.
  if (env.RERANK_TYPE === 'api') {
    return rerankExternalApi(params.query, params.candidates);
  }

  if (!model) return params.candidates.map(c => c.index);
  if (env.RERANK_TYPE === 'cross-encoder') {
    return rerankCrossEncoder(params.query, params.candidates);
  }

  if (!env.DISTILLATION_ENABLED) return params.candidates.map(c => c.index);
  return rerankGenerative(params.query, params.candidates);
}
```

## 6. Unit test strategy

Mock fetch via `global.fetch` assignment (node:test compatible pattern). Test:
1. Happy path: fetch returns `[{index:2, score:0.5}, {index:0, score:0.3}, {index:1, score:0.1}]` → function returns `[candidates[2].index, candidates[0].index, candidates[1].index]`.
2. HTTP error: returns original order.
3. Network error (fetch throws): returns original order.
4. Empty response array: returns original order.

Save original `global.fetch`, restore in `try/finally`. Match existing test patterns in `salience.test.ts`.

## 7. Baseline run protocol

**Pre-flight:**
1. `docker compose up -d tei-rerank` (first run: wait ~5min for model download).
2. Verify: `curl -s http://localhost:8080/health` returns 200. `curl -s http://localhost:8080/info` returns model info.
3. `lesson_access_log` COUNT = 90.

**Run:**
```bash
RERANK_TYPE=api LESSONS_SALIENCE_NO_WRITE=true docker compose up -d --force-recreate mcp
# verify env
docker compose exec mcp node -e "console.log('RT:', process.env.RERANK_TYPE)"
# baseline
npm run qc:baseline -- --tag sprint-12.1g-tei-bge --samples 1 --surfaces lessons
```

**Determinism check:** recreate mcp (same env), run baseline with tag `sprint-12.1g-tei-bge-repeat`. Compare per-query found_ranks: expect 0/40 diffs.

**Restore defaults:** `docker compose up -d --force-recreate mcp` (no env overrides).

## 9. Decision matrix (winner criteria per spec §A9)

| TEI+bge result | Action |
|---|---|
| MRR ≥ 0.98 AND nDCG@10 ≥ 0.96 AND deterministic | **Ship as default.** Change `RERANK_TYPE` default from `generative` to `api` in env.ts. |
| MRR ∈ [0.95, 0.98) AND deterministic | **Partial win.** Document: api is deterministic alternative; generative still wins on quality. User picks per use case. Keep generative as default. |
| Not deterministic (0/40 repeat fails) | **Reject.** Same problem as generative. Document + move on. |
| MRR < 0.95 | **Worse than gte.** Document, maybe try jina or qwen reranker on TEI next sprint. |

## 10. Risks introduced at design

**R-D1 — First `docker compose up` blocks on model download.** 5min cold start. Mitigation: start_period=300s on healthcheck prevents false-fail; document the one-time cost in SESSION_PATCH.

**R-D2 — TEI uses port 8080 on host.** If user has something else on 8080, conflict. Mitigation: port binding is dev-convenience; production uses `tei-rerank:80` via Docker network. Document; change if it conflicts.

**R-D3 — rerankBudget=15 stays the same.** TEI processes 15 candidates per query in ~1s (our CLARIFY test showed this). No change needed.
