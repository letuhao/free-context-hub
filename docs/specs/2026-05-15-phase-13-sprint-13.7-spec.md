## Implementation: status transition rules (r3 F1 fix)

`updateLessonStatus` in `src/services/lessons.ts` MUST reject these transitions:

| From | To | Action | Error |
|---|---|---|---|
| active / superseded / archived | pending-review | reject | "Use submit_for_review to transition to pending-review" |
| pending-review | superseded / archived | reject | "Resolve the active review request first (approve or return)" |
| pending-review | active | accept (this is what approve_review_request does internally) | — |
| pending-review | draft | accept (this is what return_review_request does internally) | — |
| draft → any writable | allowed | — | — |
| active → archived/superseded | allowed | — | — |

Implementation note: the rule table covers ALL transitions because the master design L275-281 enumerates only ✗ rows; everything else is implicitly ✓. The guard reads `lesson.status` (current) and the input `params.status` (target), then applies the table.
