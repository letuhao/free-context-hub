# Phase 17.3 — NLI fact-checking judge (DESIGN)

**Date:** 2026-06-19 · **Closes:** the 17.3 line of DEFERRED-039; targets DEFERRED-031
(global-surface faithfulness noise). **Decision:** user chose "Build full NLI judge"
(dedicated NLI model + sidecar + judge integration + A/B) over close-as-won't-fix.

## 1. Problem (recap)

On the `global` surface, RAGAS `faithfulness` penalizes **honest meta-claims** —
statements *about* the retrieval result ("entity X is unrelated to this substring
query", "the matches are semantically diverse") — as ungrounded, because they are not
recoverable from the contexts. Two template iterations failed to fix it at the prompt
layer, and v6==v8 scored identical global faith (0.439 vs 0.444), proving the gap is
**intrinsic to substring-search semantics**, not a template/model effect. Scope: ~10
of 152 golden rows, `global` only. Production is already correct (template @ v8). This
is a **measurement-fidelity** problem.

## 2. Hypothesis the NLI judge tests

RAGAS faithfulness asks "is claim C recoverable/inferable from context?" → a binary
grounded/ungrounded. An **NLI** model classifies (premise=context, hypothesis=claim)
into **entailment / contradiction / neutral**. The hypothesis: a hallucination is only
a **contradiction**; an honest meta-claim the context is silent on is **neutral**, not
a fabrication. So a contradiction-based faithfulness should not penalize honest
meta-claims the way RAGAS does.

**Known risk (stated up front):** NLI `neutral` is ambiguous — it covers BOTH an
honest meta-claim AND a fabricated substantive claim the context happens to be silent
on. So a lenient "neutral is fine" policy may be too forgiving. The A/B is how we find
out whether the global rows' meta-claims actually map to `neutral` (fixable) and how
much silent-fabrication leaks. Building it is the only way to answer this empirically.

## 3. Model

`cross-encoder/nli-deberta-v3-small` (HuggingFace). CPU-friendly (768 hidden), labels
`{0: contradiction, 1: entailment, 2: neutral}`. Loaded via `sentence_transformers.CrossEncoder`.
Baked into the image at build time (HF reachable; verified) so there is no runtime
download or network dependency. Env `NLI_MODEL` overrides for a future upgrade
(e.g. `nli-deberta-v3-base` for accuracy at higher CPU cost).

## 4. Service — `services/nli-judge/` (FastAPI, mirrors ragas-judge)

Internal port 8000 → host **3006** (adjacent to ragas-judge 3005). `profiles:
["measurement"]`, no LM Studio dependency (fully self-contained). Endpoints:

- `GET /health` → `{ status, model, loaded }` — cheap, does NOT run inference.
- `POST /entail` `{ premise, hypothesis }` → `{ label, scores: {contradiction,
  entailment, neutral} }` — the pure NLI primitive (softmax over the 3 logits).
- `POST /score` `{ answer, contexts: [str], options?: { claim_split? } }` →
  ```
  {
    n_claims, 
    per_claim: [{ claim, label, scores }],
    nli_faithfulness_strict,    // #entailment / n              (≈ RAGAS-like)
    nli_faithfulness_lenient,   // (#entailment + #neutral) / n  = 1 − contradiction_rate
    nli_contradiction_rate      // #contradiction / n            (clean hallucination signal)
  }
  ```
  Claim decomposition: sentence-split (regex, dependency-free) by default. An
  LLM-based statement extractor (as RAGAS uses) would be more faithful but couples the
  service to LM Studio; deferred — sentence granularity is adequate for the global
  rows' short answers. premise = contexts joined with `\n\n`; hypothesis = each claim.
  Long premises are truncated to the model's token budget (handled by the tokenizer).

**Why report all three scores:** `strict` reproduces the RAGAS penalty (control);
`lenient` is the "neutral is not fabrication" hypothesis; `contradiction_rate` is the
surface-agnostic hallucination metric that is meaningful regardless of the meta-claim
debate. The A/B compares all three against RAGAS faithfulness on the same rows.

## 5. TS integration (production-OFF)

- `src/qc/nliScore.ts` — typed client for `/entail` + `/score` (timeout + single
  retry, mirrors `judge.ts`). `NLI_JUDGE_URL` default `http://localhost:3006`.
- `judge.ts` — new `MetricName`s `nli_faithfulness_strict | nli_faithfulness_lenient
  | nli_contradiction_rate` in the non-standard union (NOT in the default sidecar
  metric set). Only computed when explicitly requested.
- `runBaseline.ts` — `--nli` flag (or `NLI_JUDGE_ENABLED=1`) adds the NLI metrics to
  the gen-eval row on the `global` surface. Default OFF → zero production/baseline
  behavior change. The metrics are **advisory** (reported, not gating).

## 6. A/B plan

Reuse an existing global-surface gen-eval archive (saved `generated_answer` +
`contexts_used` per row) so the A/B does NOT depend on re-running synthesis. For each
of the ~10 global rows: compute the 3 NLI scores and place them beside the row's RAGAS
faithfulness. Success criterion (decision gate):

- **Confirms NLI helps** iff the honest-meta-claim rows that RAGAS scored low get a
  high `lenient` / low `contradiction_rate`, AND substantive-fabrication rows still
  score badly (high contradiction). → ship NLI as the global-surface faithfulness, A/B
  documented; DEFERRED-031 RESOLVED (fixed).
- **Confirms it does NOT cleanly help** (meta-claims land in neutral but so do
  fabrications, so no policy separates them) → DEFERRED-031 RESOLVED (won't-fix,
  empirically grounded): global gen-eval faith is intrinsically noisy; use
  retrieval metrics. Either way 17.3 is CLEARED — built, measured, decided.

## 7. Non-goals / caveats

- Not wired into production chat or the default baseline; measurement-profile only.
- Sentence-split (not LLM statement extraction) — coarser but dependency-free.
- `nli-deberta-v3-small` is the accuracy floor; upgrade path via `NLI_MODEL`.
- Out of scope: applying NLI to lessons/code/chunks (they don't have this gap).
