# Spec — Phase 12 Sprint 12.1f — Cross-encoder rerank evaluation

Measurement-infrastructure sprint. Tests whether cross-encoder rerank can replace LM Studio's generative rerank (found to drift non-deterministically across sessions in Sprint 12.1e4).

## Problem

Sprint 12.1e4 proved the generative LLM reranker (`DISTILLATION_MODEL=qwen/qwen2.5-coder-14b` at temp=0) drifts ~0.027 MRR / ~0.031 nDCG@10 across container recreates despite deterministic-looking temperature. This makes it unfit for QC measurement. Current mitigation is `DISTILLATION_ENABLED=false` which skips rerank entirely — giving up ~0.06 MRR quality improvement.

The salience/retrieval code supports an alternative: `RERANK_TYPE=cross-encoder` (`src/services/lessons.ts:605`). Cross-encoder rerank uses embedding cosine similarity between query and each candidate, scored by a model trained for relevance discrimination (bge-reranker, gte-reranker, jina-reranker, etc.). This SHOULD be deterministic — embedding backends don't have temperature. But it's untested on our goldenset; we don't know if quality is comparable to generative.

LM Studio has multiple cross-encoder rerankers available: `text-embedding-bge-reranker-v2-m3`, `gte-reranker-modernbert-base`, `text-embedding-jina-reranker-v3`, `qwen3-reranker-0.6b`, `llama-nemotron-rerank-1b-v2`.

## Scope

**IN SCOPE:**
1. **3 cross-encoder models tested** on the 40q lessons goldenset:
   - `text-embedding-bge-reranker-v2-m3` — strong, well-known baseline
   - `gte-reranker-modernbert-base` — modern architecture, fast
   - `text-embedding-jina-reranker-v3` — newer, strong
2. **1 generative-rerank baseline** run today (fresh, same session) — apples-to-apples reference for cross-encoder comparison.
3. **Winner's same-config-repeat** — verify the best cross-encoder is deterministic across container recreates.
4. **Summary doc** with quality comparison (MRR, nDCG@10, per-group) + determinism verdict + recommendation.
5. **Config change (conditional):** if cross-encoder winner is ≥ generative within noise AND deterministic, recommend changing `RERANK_TYPE` default to `cross-encoder`. If not, keep `generative` and document no-rerank as the measurement-only alternative.

**OUT OF SCOPE:**
- Testing all 9+ cross-encoder rerankers in LM Studio (pick 3 strong ones; deep tuning later if promising).
- Salience math changes.
- Default changes to α / HL / salience.
- Generative rerank optimization (we know it drifts; fix is out of our hands).

## CLARIFY decisions locked

**D1 — Models tested:** bge-reranker-v2-m3, gte-reranker-modernbert-base, jina-reranker-v3. Three architecturally distinct strong rerankers.

**D2 — Reference baseline:** fresh generative-rerank run today (not reuse 12.1e4's, which had drift contamination). Apples-to-apples with cross-encoder runs in the same session.

**D3 — NO_WRITE=true on every run.** All salience writes suppressed. α=0.10, HL=7 (production defaults) — we're measuring rerank, not salience.

**D4 — Winner definition:** highest aggregate nDCG@10 on the 40q goldenset among the 3 cross-encoders. If any beats generative nDCG@10 − noise floor (0.013), candidate for default change. Otherwise best cross-encoder becomes "measurement mode" alternative to no-rerank.

**D5 — Determinism check:** winner model, same env, force-recreate container, rerun. 0/40 query `found_ranks` diffs required.

**D6 — Run order:** generative-ref → bge → gte → jina → winner-repeat. Sequential, each recreates mcp.

## Risks + mitigations

**R1 — Cross-encoder quality is significantly worse than generative.** Possible; cross-encoders are typically faster but less nuanced. Mitigation: data-driven — if all 3 are worse, don't recommend switching; document no-rerank as the measurement default.

**R2 — Model not actually loaded in LM Studio.** If one of the 3 models isn't loaded/ready when queried, rerank falls back to unchanged order. Mitigation: verify model responds via `/v1/models` and `/v1/embeddings` smoke test before each baseline run.

**R3 — Cross-encoder has its own non-determinism.** Embedding backends SHOULD be deterministic but may not be (batch effects, quantization, etc.). Mitigation: D5 determinism check verifies this empirically. If winner drifts, the cross-encoder switch is moot; same problem as generative.

**R4 — Quality comparison is confounded by α/HL salience state.** Sprint 12.1e4 showed α has zero effect on current goldenset state (bootstrap-only). So salience won't confound rerank comparison on this goldenset. But in production with real traffic, salience + rerank interact — this sprint can't measure that.

**R5 — Per-model env setup.** Cross-encoder needs `RERANK_MODEL=<model-name>` set per run. No existing docker-compose exposure for `RERANK_MODEL` or `RERANK_TYPE`. Add both as one-line exposures.
