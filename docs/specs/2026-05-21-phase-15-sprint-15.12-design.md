## ¬ß1 Part A ‚Äî `requireResourceScope` middleware (DEFERRED-009)

New `src/api/middleware/requireResourceScope.ts`. A DB-lookup tenant-scope guard
parameterized by an **entity resolver** (how to get the owning `project_id` from a
URL param).

```ts
type ScopeEntity = 'topic' | 'request' | 'motion' | 'dispute' | 'intake' | 'body' | 'task' | 'artifact';

const RESOLVERS: Record<ScopeEntity, { sql: string }> = {
  topic:    { sql: `SELECT project_id FROM topics WHERE topic_id = $1` },
  request:  { sql: `SELECT t.project_id FROM requests r JOIN topics t ON t.topic_id=r.topic_id WHERE r.request_id = $1` },
  motion:   { sql: `SELECT t.project_id FROM motions m JOIN topics t ON t.topic_id=m.topic_id WHERE m.motion_id = $1` },
  dispute:  { sql: `SELECT t.project_id FROM disputes d JOIN topics t ON t.topic_id=d.topic_id WHERE d.dispute_id = $1` },
  intake:   { sql: `SELECT project_id FROM intake_items WHERE intake_id = $1` },
  body:     { sql: `SELECT project_id FROM decision_bodies WHERE body_id = $1` },
  task:     { sql: `SELECT t.project_id FROM tasks tk JOIN topics t ON t.topic_id=tk.topic_id WHERE tk.task_id = $1` },
  // rev 2 (F2) ‚Äî artifact resolver added (was referenced in ¬ß1.3 but missing from the map)
  artifact: { sql: `SELECT t.project_id FROM artifacts a JOIN tasks tk ON tk.task_id=a.task_id JOIN topics t ON t.topic_id=tk.topic_id WHERE a.artifact_id = $1` },
};

export function requireResourceScope(entity: ScopeEntity, paramName = 'id') {
  return async (req, res, next) => {
    const attachedScope = (req as { apiKeyScope?: string | null }).apiKeyScope;
    if (attachedScope === undefined) return next(); // auth-off / env-token ‚Üí unrestricted
    if (attachedScope === null) return next();       // global scope ‚Üí any project
    const idVal = (req.params as Record<string,string>)[paramName];
    if (!idVal) { res.status(400).json({ error: `requireResourceScope: missing :${paramName}` }); return; }
    try {
      const r = await getDbPool().query(RESOLVERS[entity].sql, [idVal]);
      // 0 rows (unknown) OR project mismatch ‚Üí 404 (Q2: no cross-tenant existence disclosure)
      if (r.rowCount === 0 || r.rows[0].project_id !== attachedScope) {
        res.status(404).json({ status: 'error', error: `${entity} not found`, code: 'NOT_FOUND' });
        return;
      }
      next();
    } catch (e) { next(e); }
  };
}
```

### ¬ß1.1 Body-project routes (creation routes where project_id is in the BODY)
Two routes take `project_id` in the request body, not a URL param:
- `POST /api/decision-bodies` (createBody) ‚Äî `body.project_id`
- `POST /api/intake` (submitIntake) ‚Äî `body.project_id`

For these, a sibling guard `requireBodyProjectScope()` ‚Äî **rev 2 (F1 fix):** a scoped
key that omits the body project must NOT fall through to `DEFAULT_PROJECT_ID` at the
service (that would escape its scope). Inject the key's scope on omission:
```ts
export function requireBodyProjectScope(bodyField = 'project_id') {
  return (req, res, next) => {
    const scope = (req as {apiKeyScope?: string|null}).apiKeyScope;
    if (scope === undefined || scope === null) return next(); // auth-off / global
    const declared = (req.body ?? {})[bodyField];
    if (declared === undefined) {
      // F1 ‚Äî a scoped key's resource defaults to ITS OWN project, never DEFAULT_PROJECT_ID.
      req.body = { ...(req.body ?? {}), [bodyField]: scope };
      return next();
    }
    if (declared !== scope) {
      res.status(404).json({ status: 'error', error: 'project not found', code: 'NOT_FOUND' });
      return;
    }
    next();
  };
}
```
A scoped key always writes to its own project; an explicit cross-project declaration ‚Üí
404. Auth-off / global ‚Üí unchanged.

## ¬ß2 Part B ‚Äî induction-pack tail mode (DEFERRED-010)

### ¬ß2.1 `coordinationEvents.ts:replayEvents` ‚Äî add tail mode
Current signature: `replayEvents({ topic_id, since_seq, limit }, executor?)`. Add an
optional `tail?: boolean`:
- `tail` falsy (default): unchanged ‚Äî `WHERE seq > since_seq ORDER BY seq ASC LIMIT N`.
- `tail: true`: `ORDER BY seq DESC LIMIT N`, re-sorted ASC in JS; `next_cursor` /
  `your_cursor` = max(seq).

```sql
-- tail mode
SELECT ... FROM coordination_events WHERE topic_id=$1 ORDER BY seq DESC LIMIT $2
-- then reverse in JS to ASC.
```
N = `DEFAULT_REPLAY_LIMIT` (1000, reused per Q3).

**rev 2 (F3) ‚Äî `has_more` computation (cheap, unambiguous):**
- If `returned_count < N` ‚Üí the window covers all events ‚Üí `has_more = false` (no query).
- Else ‚Üí `has_more = EXISTS(SELECT 1 FROM coordination_events WHERE topic_id=$1 AND seq
  < $minReturnedSeq)` ‚Äî O(1) on the `(topic_id, seq)` index. NO full `COUNT(*)`.

### ¬ß2.2 `topics.ts:joinTopic` ‚Äî induction pack uses tail mode
The induction-pack replay (txn 2) currently calls `replayEvents({topic_id, since_seq})`.
Change: when the caller passes `since_seq` (re-prime from a known cursor) keep the
forward behavior; for a FRESH join (`since_seq` absent / 0), use `tail: true` so the
pack carries the most-recent events incl. the joiner's own `topic.actor_joined`.

Decision: a fresh join (`since_seq === 0`) ‚Üí tail mode; a re-prime (`since_seq > 0`) ‚Üí
forward mode (the existing cursor-continuation contract). This preserves the re-prime
path (AC12) and fixes the fresh-join ergonomics (AC10).

### ¬ß2.3 Coherence
`your_cursor` = max seq ‚Üí the joiner is primed to HEAD. `has_more=true` signals older
history exists (reachable by an explicit backward/forward replay ‚Äî not added in 15.12;
the deferred only requires recent events + has_more, both delivered).

## ¬ß6 Light security checklist (POST-REVIEW, Q4)
1. A project-A key cannot read/join/close/grant a project-B topic (404, not 200/403).
2. A project-A key cannot decide a project-B request / tally a project-B motion / etc.
3. A project-A key cannot create a body / submit intake into project B (body-project).
4. Unknown id and cross-tenant id are indistinguishable (both 404).
5. Auth-off + global-scope paths remain unrestricted (no regression).
6. No id-probing oracle (timing/error-shape) distinguishes cross-tenant from unknown.
