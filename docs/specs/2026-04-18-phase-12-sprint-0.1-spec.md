## Work item 1 — dup-rate v1

### Goal
Surface the duplication pathology the v0 metric was blind to (identical-
title-different-UUID lesson clusters AND different-title-same-snippet
fixture clusters).

### Design

Add two pure helpers to [src/qc/metrics.ts](src/qc/metrics.ts):

```ts
/** Normalize a text field for near-duplicate hashing.
 *  - lowercase
 *  - replace digit runs with a single 'N' (collapses timestamp variance:
 *    "impexp-1775368159562" → "impexp-N")
 *  - collapse whitespace runs to single spaces
 *  - trim
 */
export function normalizeForHash(s: string): string;

/** Combine title + snippet-prefix into one near-semantic key.
 *  Both fields normalizeForHash'd, separated by an unambiguous delimiter
 *  so "a||b" and "a|" + "|b" don't collide. */
export function nearSemanticKey(title: string | undefined, snippet: string | undefined): string;
```

`duplicationRateAtK` stays unchanged — already generic over `{ key: string }`.
The CALLER (aggregate() in runBaseline.ts) builds two arrays:

```ts
// v0 (existing): key = entity id (SurfaceItem.key)
const idItems = q.top_k_keys.map((k) => ({ key: k }));
const sumDupV0 = sum over queries of duplicationRateAtK(idItems, 10)

// v1 (new): key = nearSemanticKey(title, snippet)
const semItems = q.top_k_keys.map((_, i) => ({
  key: nearSemanticKey(q.top_k_titles[i], q.top_k_snippets[i])
}));
const sumDupV1 = sum over queries of duplicationRateAtK(semItems, 10)
```

Requires passing `top_k_snippets` through PerQuery → JSON archive. Today
snippets are retrieved in SurfaceItem but dropped in `runBaseline.ts:188`
(only `keys` and `titles` are stored). Add a third field.

### Expected behavior
- Cluster of 10 lessons all titled "Global search test retry pattern" with
  identical snippets → dup@10_nearsem = 1.0.
- Cluster of 10 "Valid: impexp-\<ts\>-extra" fixtures with different title
  suffixes but identical 100-char snippet prefixes → normalizeForHash
  collapses the ts → keys identical → dup@10_nearsem = 1.0.
- Distinct real content → keys distinct → dup@10_nearsem = 0.
- v0 `duplication_rate_at_10` stays 0 in all these cases (exact id unchanged).
