# Sprint 13.6 — F3 GUI: Taxonomy panel on Project Settings

## What this sprint ships

Project Settings page gets a new **Taxonomy** section between Knowledge Exchange and Danger Zone:

1. View the active profile (or empty state)
2. Activate a built-in or custom profile via picker
3. Deactivate with a confirmation dialog warning about orphaned lesson_type strings

Out of scope (deferred):
- Creating custom profiles via GUI
- Profile-aware label rendering across other pages (Lessons, Review)

## ACs

- GUI-F3-AC1: Taxonomy section visible after Knowledge Exchange
- GUI-F3-AC2: Section shows active profile or "None active"
- GUI-F3-AC3: Picker + Activate button calling REST POST /activate
- GUI-F3-AC4: Deactivate button with confirm dialog calling REST DELETE
- GUI-F3-AC5: Lesson type chips render with profile-defined color + label
