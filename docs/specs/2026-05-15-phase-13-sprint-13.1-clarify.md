## Intent

Implement Feature 1 core: artifact leasing protocol. Agents call `claim_artifact(agent_id, artifact_type, artifact_id, task_description, ttl)` to declare intent on a named artifact. Concurrent claims on same artifact → exactly one `claimed`, others `conflict`. Lease has TTL. Owner can `release_artifact` or `renew_artifact`. Anyone can `list_active_claims` or `check_artifact_availability`. Rate limit: max 10 active leases per `(agent_id, project_id)`.

## Why now

DLF (Dead Light Framework) Phase 0 audit found agents step on each other when working on the same artifact (`reckoning-record-s1` being edited by two agents in parallel). Phase 13 closes this gap. Sprint 13.1 ships the core protocol; 13.2 ships TTL sweep + GUI.

## Assumptions (locked at CLARIFY)

| # | Assumption | If wrong |

## Three approaches considered

### Approach A — Single service module + thin handlers (RECOMMENDED)

`src/services/artifactLeases.ts` implements all 5 business operations + atomic transaction logic. `src/mcp/index.ts` adds 5 tool handlers that are thin wrappers calling the service. `src/api/routes/artifactLeases.ts` adds REST handlers that are thin wrappers calling the same service.

**Pros:**
- Same business logic exposed via MCP + REST → no drift between channels
- Service has only `pg` + types as dependencies → easy to unit-test with a real test DB
- Atomic transaction lives in one place

**Cons:**
- Need to pass project_id through both channels' parameter validation layers
- Rate limit state lives in DB queries (no in-memory state) → relies on `(project_id, agent_id) WHERE expires_at > now()` index

### Approach B — Direct DB access from handlers

MCP and REST handlers each have their own DB queries. No shared service module.

**Pros:**
- Smaller file count

**Cons:**
- Duplication of complex atomic-tx logic in 2 places — high drift risk
- Harder to unit-test (need to test each handler separately)
- **Rejected**

### Approach C — In-memory leases (no DB)

Keep all leases in process memory (Map). Lose all leases on restart.

**Pros:**
- Faster (no DB roundtrip)

**Cons:**
- No persistence; restart = data loss; multi-process = leases not shared
- **Rejected** — spec explicitly requires DB persistence

**Recommendation: A.**

## Risks

| # | Risk | Severity | Mitigation |
|---|---|---|---|
| R1 | Concurrent INSERT race: 2 agents pass step 3 (SELECT) and both INSERT (step 4). Partial unique index catches one with `23505` violation. | HIGH | Catch `23505` and map to `conflict` response. Add concurrent-claim test. |
| R2 | Rate limit COUNT not atomic with INSERT — agent could exceed 10 leases by racing | MED | Acceptable for Phase 13: COUNT inside same TX = at-most-one-extra-race. Document as known limitation. |
| R3 | Lazy cleanup races with another agent's claim — if agent A deletes B's expired lease while B is renewing, B's renew fails. | MED | Renew specifically checks `expires_at > now()` BEFORE doing the cleanup. If expired at check time, return `expired`. Owner can re-claim. |
| R4 | `agent_id` collision: two agents pick same string | LOW | Acceptable per spec D1 |
| R5 | `task_description` injection (XSS in GUI) | MED | GUI is Sprint 13.2 scope; will escape on render. Service stores as raw text. |
| R6 | Admin force-release endpoint without admin middleware = anyone can release any lease | HIGH | Verify admin middleware exists OR add TODO. If missing, route returns 501 until middleware added. |

## Acceptance criteria (Sprint 13.1 only)

- [ ] AC1: Migration 0048 applies cleanly (idempotent if re-run)
- [ ] AC2: `claim_artifact` returns `{status: 'claimed', lease_id, expires_at}` when available
- [ ] AC3: `claim_artifact` returns `{status: 'conflict', incumbent_agent_id, expires_at, seconds_remaining}` when lease exists
- [ ] AC4: `claim_artifact` returns `{status: 'rate_limited'}` when agent has ≥10 active leases
- [ ] AC5: Concurrent `claim_artifact` calls on same artifact: exactly one returns `claimed`, all others return `conflict` (NOT 500) — covered by unit test with 2+ simultaneous claims
- [ ] AC6: `release_artifact` returns `{status: 'released'}` when caller is owner; `not_owner` if agent_id mismatch; `not_found` if lease_id unknown
- [ ] AC7: `renew_artifact` extends `expires_at`; returns `expired` if lease already past expiry; `not_owner` if agent_id mismatch
- [ ] AC8: `list_active_claims` returns only non-expired leases; supports optional `artifact_type` filter
- [ ] AC9: `check_artifact_availability` returns `{available: true}` if no active lease; otherwise `{available: false, lease: {...}}`
- [ ] AC10: REST endpoints mirror MCP tools 1:1; each endpoint enforces `project_id` from URL path
- [ ] AC11: Admin DELETE `/api/admin/artifact-leases/:leaseId` requires admin role (or returns 501 if middleware unavailable, with TODO)
- [ ] AC12: All unit tests pass; `tsc --noEmit` clean
- [ ] AC13: Manual MCP smoke: `claim` → `list_active_claims` → `release` → `list_active_claims` returns empty
